# GitHub Basics

## What is GitHub?
GitHub is a platform that uses Git to store, manage, and track changes in projects.

---

## What is Git?
Git is a version control system.

It records every change made to files so we can:
- Go back to previous versions
- See what changed and why
- Work safely without losing progress

Git is a timeline of our work.

---

## Key Concepts

### Repository (Repo)
A repository is a project workspace.

When do we use it?
- When starting a new project
- When organizing files (code, notes, simulations)
- When tracking changes over time

---

### Commit
A commit is a saved version (snapshot) of a project.

When do we use it?
- After making meaningful changes
- To record progress
- Before trying something risky

What happens after a commit?
- Changes are saved permanently
- A new version is added to the project history
- Previous versions remain accessible

---

### Branch
A branch is a separate version of a project.

Why and when do we use it?
- To test new ideas without affecting the main project
- To work on features independently
- To experiment safely

Example:  
Main branch → stable version  
New branch → testing a new idea

## Practice

### I created my first repository: `Learning-github`

How?
- Clicked "Create repository" on GitHub
- Named the project
- Initialized it with a README file

#### What is a README file?
A `README.md` is the front page of a project.

It serves as:
- A project introduction
- A guide for anyone opening the repository (including future me)

It explains:
- What the project is
- Why it exists
- What it contains

---

### I edited the README

Why?
- To explain the purpose of the project
- To document learning

How?
- Clicked the edit button
- Modified the content
- Committed changes

When?
- At the start of a project
- Whenever the project evolves

---

### I made commits

What happens after a commit?
- Changes are saved as a new version
- A history of progress is created
- The project becomes traceable and structured
